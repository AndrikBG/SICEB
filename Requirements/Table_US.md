##   USER STORIES

| ID | User Story (As a... I want... so that...) | Priority |
|----|-----------------------------------------------------|-----------|
| **US-001** | As an **Administrator**, I want to create user accounts with role-based permissions so that staff can access the system according to their responsibilities | **CRITICAL** |
| **US-002** | As a **system user**, I want to log in with my credentials to access the system securely | **CRITICAL** |
| **US-003** | As an **Administrator**, I want to assign different permission levels (Director, Admin, Manager, Physician, Resident, etc.) so that users only see information relevant to their role | **CRITICAL** |
| **US-004** | As a **General Administrator**, I want to see the complete inventory of ALL services to have total visibility of clinical resources | **CRITICAL** |
| **US-005** | As a **Service Manager**, I want to see ONLY the inventory of my service to manage the resources of my area effectively | **CRITICAL** |
| **US-006** | As a **Service Manager**, I want to see automatic alerts when inventory items are running low to request restocking before running out of stock | **HIGH** |
| **US-007** | As a **Service Manager**, I want to see alerts for items close to expiration to use them before they expire and minimize waste | **MEDIUM** |
| **US-008** | As **Lab Personnel**, I want to receive alerts when reagents are outside the safe temperature range to take corrective action immediately | **HIGH** |
| **US-009** | As a **Service Manager**, I want to send a digital supply request so that the process is faster and traceable | **HIGH** |
| **US-010** | As a **General Administrator**, I want to receive automatic notifications when supply requests are sent to review them promptly | **HIGH** |
| **US-011** | As a **General Administrator**, I want to approve or reject supply requests with justification so that decisions are documented | **HIGH** |
| **US-012** | As a **Service Manager**, I want to receive automatic notifications when my request is approved or rejected to know the status without manual follow-up | **MEDIUM** |
| **US-013** | As a **General Administrator** and **Service Manager**, we want to complete the material delivery cycle (Admin records delivery → Manager confirms receipt → Inventory updates) to close the request flow with complete traceability | **MEDIUM** |
| **US-015** | As a **Service Manager** or **Resident**, I want to send a digital workshop/training request so that the approval process is formalized | **MEDIUM** |
| **US-016** | As a **General Administrator**, I want to approve or reject workshop requests so that training activities are controlled | **MEDIUM** |
| **US-017** | As a **General Administrator**, I want to see the history of all workshop requests to track training activity | **LOW** |
| **US-018** | As a **Service Manager**, I want to register attendance of completed workshops so that resident participation is documented | **MEDIUM** |
| **US-019** | As **Reception Personnel**, I want to register new patients with their demographic information so they can receive medical attention | **CRITICAL** |
| **US-020** | As **Reception Personnel**, I want to classify patients by type (Student/Worker/External) AND have the system automatically apply the corresponding discount (30% Students, 20% Workers, 0% External) so that billing is consistent and automatic | **CRITICAL** |
| **US-022** | As **Reception Personnel**, I want to register a minor patient with their guardian's information to align with legal requirements | **HIGH** |
| **US-023** | As **Reception Personnel**, I want the system to validate that a guardian is present for patients under 17 years old to enforce clinic policies | **CRITICAL** |
| **US-024** | As a **Physician** or **Service Manager**, I want to create a new medical record when I register a patient to start their medical history | **CRITICAL** |
| **US-025** | As a **Physician**, I want to add a new consultation entry to a patient's existing record to document their care | **CRITICAL** |
| **US-026** | As a **Physician**, I want consultation entries to be immutable (not editable) so that medical records maintain integrity for audits | **CRITICAL** |
| **US-027** | As a **Physician**, I want to see the comprehensive medical history of a patient from all services to have full context for treatment decisions | **CRITICAL** |
| **US-028** | As a **Physician**, I want to search for patients by name, ID, or student/employee number to quickly access their records | **HIGH** |
| **US-029** | As a **Physician**, I want to attach files (PDFs, images) to a patient's record so that external documents are consolidated | **HIGH** |
| **US-030** | As a **Physician**, I want to register vital signs during a consultation so that patient health trends can be tracked | **HIGH** |
| **US-031** | As a **Physician**, I want to prescribe medications during a consultation so that the prescription is recorded and available for pharmacy | **CRITICAL** |
| **US-032** | As **Pharmacy Personnel**, I want to see a patient's prescriptions to dispense the correct medications | **CRITICAL** |
| **US-033** | As **Pharmacy Personnel**, I want the system to validate that a prescription exists before dispensing to not dispense without a medical order | **CRITICAL** |
| **US-034** | As **Pharmacy Personnel**, I want the system to verify inventory before dispensing to know if medications are available | **CRITICAL** |
| **US-035** | As **Pharmacy Personnel**, I want to register when I dispense controlled medications for complete traceability | **CRITICAL** |
| **US-036** | As **Pharmacy Personnel**, I want to register the dispensation with who prescribed, who dispensed, and to whom for a complete audit trail | **HIGH** |
| **US-037** | As **Pharmacy Personnel**, I want to charge for medications separately from the consultation so that accounting is exact | **HIGH** |
| **US-038** | As a **Physician**, I want to request laboratory studies during a consultation to order diagnostic tests | **CRITICAL** |
| **US-039** | As **Reception Personnel**, I want to charge for laboratory studies BEFORE performing them to ensure payment in advance | **HIGH** |
| **US-040** | As **Lab Personnel**, I want to see pending study requests to know which tests to process | **CRITICAL** |
| **US-041** | As **Lab Personnel**, I want to enter study results in text format so that physicians can review them | **CRITICAL** |
| **US-042** | As a **Physician**, I want to see laboratory results in the patient's medical record to interpret them within context | **CRITICAL** |
| **US-043** | As **Lab Personnel**, I want to manage reagent inventory with expiration dates and temperature monitoring so that test quality is maintained | **HIGH** |
| **US-044** | As **Reception Personnel**, I want to register payments for consultations, medications, and laboratory studies so that all income is tracked | **CRITICAL** |
| **US-045** | As **Reception Personnel**, I want the system to generate simple receipts (not CFDI invoices) so that patients have proof of payment | **HIGH** |
| **US-046** | As a **Director** or **Administrator**, I want to generate financial reports showing income by service, concept, and patient type to analyze profitability | **HIGH** |
| **US-047** | As a **Director** or **Administrator**, I want to generate reports showing expenses by service to control costs | **HIGH** |
| **US-048** | As a **Director**, I want to see profitability reports (income - expenses) by service to make strategic decisions | **HIGH** |
| **US-049** | As an **Administrator**, I want to register medical personnel (attending and residents R1-R4) so that staff is in the system | **HIGH** |
| **US-050** | As the **system**, I want to automatically validate that residents can only perform actions allowed for their level (R1-R4) to enforce clinic policies | **CRITICAL** |
| **US-051** | As the **system**, I want to block R1, R2, and R3 Residents from prescribing controlled medications to enforce security policies | **CRITICAL** |
| **US-052** | As a **Physician**, I want to register which supplies I used during a consultation so that inventory updates and costs are tracked | **HIGH** |
| **US-053** | As **Reception Personnel**, I want to schedule medical appointments for patients specifying date, time, physician, and consultation type so that the consultation flow is organized | **HIGH** |
| **US-054** | As a **Physician**, I want to see my appointment agenda for the day with patient information to know which patients I will attend and prepare appropriately | **HIGH** |
| **US-055** | As **Reception Personnel**, I want to cancel or reschedule medical appointments documenting the reason to manage patient changes | **HIGH** |
| **US-056** | As **Administrative Personnel**, I want to register signed informed consents by attaching a scanned document to comply with NOM-024-SSA3-2012 | **HIGH** |
| **US-057** | As a **Physician**, I want to mark in the record that a procedure requires specific consent to document legal compliance | **HIGH** |
| **US-058** | As a **Service Manager**, I want to register discarded materials with a reason (expired, damaged) to document losses | **MEDIUM** |
| **US-059** | As a **Pediatrician**, I want to register growth data and vaccination schedule in pediatric records for proper follow-up | **HIGH** |
| **US-060** | As a **Gynecologist**, I want to register pregnancy data (LMP, weeks of gestation, EDD) for prenatal follow-up | **HIGH** |
| **US-061** | As **Emergency Personnel**, I want to assign triage level (Red/Yellow/Green) to prioritize care according to severity | **HIGH** |
| **US-062** | As a **Patient**, I want to request access to my complete medical record (ARCO right) to exercise my rights | **HIGH** |
| **US-063** | As **Administrative Personnel**, I want to process ARCO requests within the legal timeframe (20 days) to comply with LFPDPPP | **HIGH** |
| **US-064** | As an **Administrator**, I want to configure tariffs by medical service specifying base price so that the system calculates charges correctly | **CRITICAL** |
| **US-065** | As an **Administrator**, I want to assign a supervisor physician to R1/R2 residents to comply with mandatory supervision policies | **HIGH** |
| **US-066** | As the **system**, I want to register in an audit log who accessed which record and when to comply with LFPDPPP traceability | **CRITICAL** |
| **US-067** | As a **Director**, I want to see an executive dashboard with key metrics to monitor clinic performance | **MEDIUM** |
| **US-068** | As an **Administrator**, I want to generate a report of supply requests by service and period to analyze consumption patterns | **MEDIUM** |
| **US-070** | As the **system**, I want to send appointment reminders 24h before to reduce absenteeism | **LOW** |
